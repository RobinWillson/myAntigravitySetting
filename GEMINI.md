# Antigravity Global Rules (全域安全與程式碼規則)

本規則為電腦全域生效（Global Scope）之最高原則。Agent 在與使用者進行對話、分析代碼或執行終端機指令時，必須無條件嚴格遵守以下底層約束：

---

## 🛡️ Python 與系統命令執行安全規範

為保障使用者作業系統的穩定性與隱私，防止誤觸高風險指令或搞亂全域開發環境，必須遵守以下限制：

### 1. 嚴禁全域系統級安裝與寫入 (No Global System Python Writes)
- **禁止使用危險參數**：嚴禁自動執行或向使用者提議執行帶有 `--break-system-packages` 參數的 `pip` 或 `pip3` 安裝命令。
- **禁止污染全域環境**：嚴禁在無 Python 虛擬環境（`venv`）保護的情況下，直接向全域系統路徑安裝任何第三方套件。
- **禁止濫用管理員權限**：非極端必要且有明確系統維護需求，嚴禁使用 `sudo` 執行 Python 安裝、環境配置或未知腳本。

### 2. 強制使用虛擬環境沙盒 (Mandatory Virtual Environments)
- **虛擬環境優先**：凡涉及任何 Python 專案的依賴套件安裝、工具測試或 CLI 執行，Agent 必須優先引導使用者建立或在背景建立 `venv` / `conda` 沙盒環境。
- **隔離依賴**：所有安裝行為必須完全侷限在專案本地的虛擬環境資料夾內，確保全域環境的乾淨。

### 3. 執行前必先說明與獲取明確授權 (Explain & Verify Before Command Run)
- **白話文事前說明**：在向使用者提議或執行任何涉及「環境配置變更」、「環境變數修改」、「全局/局部套件安裝」或「系統級寫入」的指令前，Agent 必須先用白話文向使用者說明：
  1. *這個指令具體會做什麼事情？*
  2. *為什麼必須執行它才能解決問題？*
  3. *它對系統或目前的專案環境會有什麼潛在影響？*
- **拒絕黑箱操作**：嚴禁在使用者不理解或不知情的情況下偷偷執行、或是含糊帶過具有風險的系統環境命令。

### 4. 嚴格防範危險系統命令 (Prevent High-Risk System Commands)
- **防範指令誤傷**：嚴禁提議或執行任何可能導致系統檔案損壞、崩潰或安全性漏洞的命令，包含但不限於：
  - 危險的全域檔案權限修改（如全域 `chmod 777` 或 `chmod -R 777`）。
  - 未經確認的刪除指令（如 `rm -rf` 連帶高風險目錄或萬用字元）。
